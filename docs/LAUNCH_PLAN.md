# Launch plan — KitchenOS

## Goals

- Ship a **credible demo** for outbound and Product Hunt.
- Convert **10–30 design partners** with feedback loops.
- Keep production deployments boring (Vercel + managed Postgres).

## Timeline (suggested)

1. **Week 1–2**: Demo hub + onboarding polish + Order hub MVP drawer (if feasible).
2. **Week 3**: Beta cohort (5 kitchens) — observe Order hub + production daily.
3. **Week 4**: Product Hunt + LinkedIn sprint using scripts in `COLD_OUTREACH_SCRIPTS.md`.

## Channels

- LinkedIn founder DM + comment-led inbound.
- Cold email to ops managers at meal prep brands (small lists, high relevance).
- Partner posts from POS / Woo agencies.

## Assets

- `/demo` screen recording (90s).
- One vertical landing (`/solutions/meal-prep`) as canonical story.
- Pricing page aligned with `PRICING_STRATEGY.md`.

## Metrics

- Activation: completed onboarding + ≥1 menu + ≥1 order or demo import.
- WAU during beta; churn reason tagging in Notion/Linear.

## Risks

- Over-promising live Uber/Woo without keys → always label simulated demos.
- Performance on packing PDF on mobile → monitor Core Web Vitals.
