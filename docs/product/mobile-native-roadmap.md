# Mobile / PWA / Native Roadmap

## Current State

OS Kitchen already has:

- web app manifest
- service-worker registration
- handheld / operator web surfaces

OS Kitchen does not yet have:

- native iOS operator app
- native Android operator app

## Recommended Strategy

### Short term

Invest in PWA/operator web first:

- push notifications
- offline-aware UX
- background sync where safe
- touch target audit
- tablet ergonomics for POS/KDS

### Medium term

Define which workflows truly need native:

- POS hardware integrations
- background device permissions
- camera / scanner workflows
- push reliability

### Long term

Native apps should exist only if they materially improve:

- operator speed
- hardware ergonomics
- offline tolerance
- store distribution trust

## Positioning Rule

Today OS Kitchen should be positioned as:

- strong operator web + PWA foundation

Not as:

- fully native app suite
