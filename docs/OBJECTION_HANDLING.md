# Objection handling

| Objection | Response |
|-----------|----------|
| “We already have Shopify.” | “Shopify handles storefronts; KitchenOS handles **kitchen execution** — quantities, stations, packing, multi-brand queues.” |
| “Too busy to switch.” | “Pilot stays read-only on ingest until you trust sync — start with manual rows + one webhook.” |
| “Uber integration?” | “Uber Eats / Direct require partner credentials — we label simulated vs live clearly; Team tier covers architecture.” |
| “Security?” | Point to `/legal/security`, encryption key requirement, server-side service role usage. |
| “Price?” | Anchor ROI on mistake reduction + labor hours; offer beta concierge if applicable. |
| “Need API.” | Enterprise track — capture requirements; don’t promise timelines prematurely. |
