# Report export history

The Reports Center reuses the existing `ExportJob` table — no new model is
required. Reports Center writes records with `type = report:<reportKey>`
so we can distinguish them from legacy `/api/export?type=…` rows when
needed.

## Fields surfaced

| UI label | `ExportJob` field |
|----------|-------------------|
| When | `createdAt` |
| Report | `type` (resolved to registry title) |
| File | `fileName` |
| Rows | `rowCount` |
| Generated by | `createdBy.email` |

## Endpoints that write history

- `/api/export?type=…` (legacy) — already wrote `ExportJob` via
  `recordExportJob`. Unchanged.
- `/api/export/report?key=…` (new) — writes `ExportJob` via
  `recordReportExport`.
- `exportReportCsvAction` server action — same `recordReportExport` path.

## UI

`/dashboard/reports/history` displays the latest 100 entries sorted by
`createdAt desc`. Empty state offers a CTA back to the report library.
