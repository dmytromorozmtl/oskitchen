# Product excellence — UX final pass

## Objective

Remove “unfinished product” signals from **customer-facing** copy: avoid “placeholder”, “planned”, “ships next”, “future” unless clearly marked developer/internal.

## Patterns

- **Page headers** — title + one-line value prop + primary CTA.  
- **Empty states** — actionable next step + link to settings/integration.  
- **Tables** — sticky header, column priority on mobile, sensible defaults.  
- **Errors** — what happened, what to do, support link.  
- **Toasts** — short success; long failures as modal/detail.  
- **Confirmations** — destructive actions restate consequence.

## Scope control

- Prefer copy + layout tweaks over risky refactors.  
- Align tone with FoodOps seriousness (calm, precise, not hypey).
