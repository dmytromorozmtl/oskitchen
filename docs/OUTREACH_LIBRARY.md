## Demo script (20 min)

1. **Context (2 min):** channels, weekly volume, biggest fire drill.  
2. **Order Hub (5 min):** unified queue, statuses, channel tags.  
3. **Production (5 min):** tasks, batches, handoff from confirmed orders.  
4. **Packing (4 min):** exports / verification pattern your pilot uses.  
5. **Integrations (3 min):** webhook posture, connection health.  
6. **Close (1 min):** pilot success criteria, onboarding call, beta agreement.

## Objection: “We already use spreadsheets.”

Totally fair — spreadsheets work until two people update the same tab. KitchenOS is the system of record when orders need to become kitchen tasks and packing outputs with an audit trail.

---

## Objection: “We’re not technical.”

You shouldn’t need to be. The pilot path is: connect one channel or enter manual orders, run one full weekly cycle in demo or sandbox, then go live with founder support.
