# OS Kitchen Investor Readiness Pack

## Product Summary

OS Kitchen is a vertical SaaS operating system for food operators that combines:

- storefront
- POS
- KDS
- production planning
- packing
- routes
- CRM
- billing
- integrations
- analytics

Its strongest differentiation is not one module alone, but the unified
operator workflow across order capture, kitchen execution, fulfillment, and
customer management.

## Current Strengths

- unusually broad end-to-end food-ops surface
- strong storefront ownership
- real production / packing / routing advantage
- materially improved production hardening and readiness gates
- honest product-readiness posture is now possible through explicit matrices and audits

## Current Risks

- some roadmap-adjacent modules can still be overread by GTM if not governed
- tenant/workspace migration cleanup is not fully finished
- enterprise-comparison gaps remain in HACCP, GL, payroll, and native mobile
- proof systems for case studies / claims / KPI storytelling are still maturing

## What Has Been Completed

- canonical order write architecture is documented and enforced
- order PII encryption coverage is strong and test-backed
- API route classification now has static audit coverage
- production rate-limit and webhook-queue readiness are explicit
- cron reconciliation and workspace-scope audits exist

## What Is Pending

- deeper tenant alias cleanup
- lifecycle matrix and impersonation audit testing
- stronger productized admin replay/DLQ UX
- richer AI scheduling and food-safety depth
- proof density for public case studies and KPI claims

- multi-surface food operator

## KPI Targets

- MRR: prove repeatable early expansion, not just logo count
- active operators: show weekly operator engagement, not just accounts created
- activation rate: first operational workflow live within 14 days
- churn: keep early logo churn visibly low and explainable
- checkout uptime: target 99.9%+
- p99 latency: maintain explicit target on checkout-critical paths
- support tickets: measure by active operator and by launch cohort
- onboarding completion: make business-type-specific completion visible
