# Support & Customer Success (Final)

## Surfaces

- Customer: `/dashboard/support` — create/view/reply; no platform secrets.  
- Platform: `/platform/support`, `/queue`, `/escalations`, ticket detail.

## Status model (direction)

Align ticket statuses with enterprise practice: NEW → TRIAGED → IN_PROGRESS → WAITING_ON_CUSTOMER / WAITING_ON_SUPPORT → RESOLVED → CLOSED (+ ESCALATED path).

## Customer success signals (roadmap-friendly)

Surface cards when: **open tickets**, **failed webhooks**, **integration ERROR**, **onboarding checklist incomplete**, **usage drop** — only when metrics exist; placeholders must be labeled **beta**.

## P1

- Typed links from ticket → workspace, order, POS transaction, import batch.
