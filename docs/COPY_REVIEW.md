# Copy review — OS Kitchen

Voice for OS Kitchen: **clear, confident, operations-first, practical**. Owners and kitchen leads should know what a screen does and what to do next.

## Principles

- Prefer concrete verbs: *Connect*, *Import*, *Schedule*, *Print*, *Resolve*.
- Explain *why* when asking for permissions or integration access.
- Label simulated or demo integrations explicitly (no pretending live channels).

## Patterns

| Context | Guidance |
|---------|----------|
| Empty states | One-line outcome + primary CTA + secondary doc link |
| Errors | What failed, what to try, link to help if recurring |
| Billing | Plain-language plan limits; link to upgrade or contact |
| Onboarding | Short steps; show progress; avoid jargon |

## Audit status

- Help center pages (`/help/*`) use consistent section headings.
- Broader pass needed on integration wizards and CRM edge cases — track in `docs/STABILIZATION_AUDIT.md` P2/P3 items.

## Avoid

- “AI magic” hype, vague ROI claims, fake compliance language.
